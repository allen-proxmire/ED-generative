## §5 Update protocol

When uploading a new entry to Zenodo:

1. Generate .tex from corpus markdown via pandoc (see `Desktop/ED_tex_files/` workflow)
2. Compile to PDF via pdflatex (2 passes for cross-references)
3. Upload to Zenodo with public-facing description (see prior style: Whitepaper / Cos_05 / Paper_038_5 / GW_00 abstract refactors)
4. **Add entry to this file** with: date, title, type, version, corpus source path, brief note

When refreshing an existing Zenodo entry to v2:

1. Apply corpus-side edits to the markdown source
2. Regenerate .tex + .pdf as above
3. Upload as new version (v2.0, v3.0, etc.) to existing Zenodo entry
4. **Update version field in this file** with refresh note
